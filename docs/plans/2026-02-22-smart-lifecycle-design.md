# Smart Lifecycle + Silent WebSocket

## Problem

1. Media keys and AUHAL audio input run continuously, even when not in a Meet call
2. Electron polls the extension periodically to check call state (pull-based)
3. WebSocket reconnection attempts spam `ERR_CONNECTION_REFUSED` in browser console when Electron is offline

## Design

### Push-based Call State

Replace Electron's polling with push-based notifications from the extension.

**Current flow (pull):**
```
Electron (setInterval 3s) → queryMeetStatus → Extension → response
```

**New flow (push):**
```
Extension detects call state change → sends meet_status via WebSocket → Electron reacts
```

The content script already detects call state via MutationObserver + polling. The background script already forwards `status_changed` to the WebSocket. The missing piece: Electron reacting to these push messages to control the lifecycle.

### Reactive Lifecycle in Electron

When Electron receives `meet_status` with `active: true` (transition from inactive):
- Start media key listening (`mediaKeys.start()`)
- Start AUHAL audio input (`startAudioInput()`)

When Electron receives `meet_status` with `active: false` (transition from active):
- Stop media key listening (`mediaKeys.stop()`)
- Stop AUHAL audio input (`stopAudioInput()`)

**Safety net:** If WebSocket disconnects while in a call, assume call ended and stop everything. On reconnect, send one `query_meet_status` to sync state.

**Remove:** The periodic `setInterval` polling in `index.ts`.

### Silent WebSocket Reconnection

The `ERR_CONNECTION_REFUSED` error is generated by the browser when `new WebSocket()` fails — it cannot be caught or suppressed in JavaScript.

**Solution:** Probe with `fetch()` before attempting WebSocket connection.

```
Reconnect timer (10s) → fetch('http://127.0.0.1:18432') → failed? skip : connectWS()
```

`fetch()` failures are silent (caught by `.catch()`), while `new WebSocket()` failures always log to console. This eliminates the console spam entirely.

Additional changes:
- Increase reconnect interval from 5s to 10s
- Keep `onerror` handler empty (let `onclose` handle cleanup)

### Message Flow Summary

```
Extension (content.ts):
  DOM change detected → status_changed → background.ts

Extension (background.ts):
  status_changed received → ws.send({ type: 'meet_status', active, muted })

Electron (index.ts):
  bridge.on('meet-status') → if active changed → start/stop lifecycle

Electron (on WS disconnect):
  → stop lifecycle (safety net)

Electron (on WS reconnect):
  → query_meet_status once → sync state
```

### Error Handling

| Scenario | Behavior |
|----------|----------|
| Electron offline | Extension probes silently, no console errors |
| Extension crashes mid-call | Electron sees WS disconnect → stops lifecycle |
| Meet tab closes | Content script stops → background sends active:false |
| Multiple Meet tabs | Existing logic: tracks per-tab, reports aggregate |
